6.2 Solver-Aided Libraries

In principle, solver-aided programming requires only symbolic values and the basic constructs described in Chapter 3. In practice, however, it is often convenient to work with richer constructs, which are built on top of these primitives. Rosette ships with two libraries that provide such constructs, as well as utility procedures for turning the results of synthesis and debugging queries into code.

6.2.1 Synthesis Library

 (require rosette/lib/meta/meta) package: base

syntax

(??)

Introduces an integer hole into a program—a placeholder for a concrete integer constant. Chapter 2.3.3 shows an example of using integer holes to sketch a factored polynomial function, which is then completed with the help of a synthesize query. The (??) construct creates and returns a fresh symbolic constant of type number?.

syntax

(choose expr ...+)

Introduces a choice hole into a program—a placeholder to be filled with one of the given expressions. This construct defines n-1 fresh boolean constants and uses them to conditionally select one of the n provided expressions.

Examples:

> (define (div2 x) ([choose >> >>> << + - *] x (??)))
> (define-symbolic i number?)
> (print-forms
   (synthesize #:forall (list i)
               #:assume (assert (>= i 0))
               #:guarantee (assert (= (div2 i) (quotient i 2)))))

'(define (div2 x) (>> x 1))

syntax

(define-synthax (id arg ...) maybe-guard body)

 
maybe-guard = 
  | #:assert guard
Defines a grammar of expressions that can be used to fill holes of the form (id expr ...). That is, writing (id expr ...) introduces a hole that is to be filled with an expression from the id grammar.

Examples:

; Defines the following grammar:
;  shift := terminal ... | const | (op shift shift)
;  op := >> | << | >>>
;  const := (??)
> (define-synthax (shift terminal ... k)
    #:assert (>= k 0)
    [choose
     terminal ... (??)
     ([choose >> << >>>] (shift terminal ... (- k 1))
                         (shift terminal ... (- k 1)))])
; A sketch with a hole to be filled with a shift expression of depth <= 2.
> (define (div2mul4 x) (shift x 2))
> (define-symbolic i number?)
> (print-forms
   (synthesize #:forall (list i)
               #:assume (assert (>= i 0))
               #:guarantee (assert (= (div2mul4 i) (* 4 (quotient i 2))))))

'(define (div2mul4 x) (<< (>>> x 1) 2))

Recursive grammars, such as shift, must be equipped with a guard that limits the size of a hole expression drawn from the grammar. Since define-synthax uses macros to implement recursive grammars, instantiating a recursive grammar with a large limit (e.g., k > 3) can cause long compilation times. The define-synthax construct may be changed in the future to a more efficient procedure-based implementation.

procedure

(generate-expressions solution)

 → (listof (cons/c syntax? syntax?))
  solution : solution?
Given a satisfiable solution? to a synthesize query, returns a list that associates each hole involved in the query with a synthesized expression. Hole completions can only be generated for programs that have been saved to disk. In the following example, generate-expressions returns a list that associates the choose hole (line 1, column 19) with the expression >>, and the ?? hole (line 1, column 46) with the expression 1.

Examples:

> (define (div2 x) ([choose >> >>> << + - *] x (??)))
> (define-symbolic i number?)
> (generate-expressions
   (synthesize #:forall (list i)
               #:assume (assert (>= i 0))
               #:guarantee (assert (= (div2 i) (quotient i 2)))))

'((#<syntax:1:19 choose> . #<syntax >>>) (#<syntax:1:46 ??> . #<syntax 1>))

procedure

(generate-forms solution) → (listof (cons/c syntax? syntax?))

  solution : solution?
Given a satisfiable solution? to a synthesize query, returns a list that associates each top-level sketch involved in the query with a completion of that sketch. Sketch completions can only be generated for programs that have been saved to disk. In the following example, generate-forms returns a list that associates the div2 sketch (line 2, column 1) with its synthesized completion.

Examples:

> (define (div2 x) ([choose >> >>> << + - *] x (??)))
> (define-symbolic i number?)
> (generate-forms
   (synthesize #:forall (list i)
               #:assume (assert (>= i 0))
               #:guarantee (assert (= (div2 i) (quotient i 2)))))

'((#<syntax:2:1 define> . #<syntax (define (div2 x) (>> x 1))>))

procedure

(print-expressions solution) → void?

  solution : solution?

procedure

(print-forms solution) → void?

  solution : solution?
Pretty-prints the result of applying generate-expressions or generate-forms to the given solution.

6.2.2 Debugging Library

 (require rosette/lib/tools/render) package: base

procedure

(render solution [font-size]) → pict?

  solution : solution?
  font-size : natural/c = 16
Given an unsatisfiable solution? to a debug query, returns a pict? visualization of that solution. The visualization displays the debugged code, highlighting the faulty expressions (i.e., those in the solution’s minimal core) in red. The optional font-size parameter controls the size of the font used to typeset the code. Visualizations can only be constructed for programs that have been saved to disk. See Chapter 2.3.2 for an example of using render.
