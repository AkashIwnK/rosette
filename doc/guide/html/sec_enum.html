5.2 Enumerations

An enumerated datatype is a type consisting of an ordered set of labeled concrete elements. Enumerated types also contain symbolic values. A symbolic value of an enumerated type evaluates to one of its concrete elements under a solution? returned by a solver-aided query. Like primitive datatypes, enumerated types include symbolic constants, which can be created using define-symbolic or define-symbolic*.

syntax

(define-enum id labels)

 
  labels : list?
Creates an enumerated type id? consisting of elements that are labeled with the given list of labels. The label values must be distinct according to equal?, and they must be immutable. Elements of the resulting type are ordered according to the labels list, so that the ith element has the ith label.

Elements of id? are recognized by the predicate id?, and they are ordered by the predicate id<?. The identifer id is a bound to a procedure that takes as input a label and returns the corresponding enum element.

Examples:

> (define-enum suit '(club diamond heart spade))
> (suit 'club)

(suit club)

> (suit? (suit 'club))

#t

> (suit<? (suit 'diamond) (suit 'heart))

#t

> (define-symbolic s suit?)
> (define env (solve (assert (suit<? s (suit 'diamond)))))
> (evaluate s env)

(suit club)

> (suit "club")

suit: contract violation

  expected: a suit label

  given: "club"

5.2.1 Generic Operations on Enumerated Datatypes

Rosette provides the following generic procedures for operating on enum types and elements:

procedure

(enum? t) → boolean?

  t : any/c
Returns true iff t is a concrete predicate that recognizes memebers of an enumerated datatype.

Examples:

> (define-enum suit '(club diamond heart spade))
> (enum? suit?)

#t

> (enum? number?)

#f

> (define-symbolic b boolean?)
> (enum? (if b suit? number?))

#f

procedure

(label element) → any/c

  element : any/c
Returns the label of the given (concrete or symbolic) enum element, or throws an error if the given value is not an element of an enumerated datatype.

Examples:

> (define-enum rgb '(red green blue))
> (label (rgb 'green))

'green

> (define-symbolic c rgb?)
> (label c)

{[(=? (rgb red) c) red] [(=? (rgb green) c) green] [(=? (rgb blue) c) blue]}

> (label "green")

label: contract violation

  expected: enum? element

  given: "green"

procedure

(ordinal element) → natural/c

  element : any/c
Returns the ordinal of the given (concrete or symbolic) enum element, or throws an error if the given value is not an element of an enumerated datatype.

Examples:

> (define-enum rgb '(red green blue))
> (ordinal (rgb 'green))

1

> (define-symbolic c rgb?)
> (ordinal c)

(bitwise-ior (ite (=? (rgb blue) c) 2 0) (ite (=? (rgb green) c) 1 0))

> (ordinal "green")

ordinal: contract violation

  expected: enum? element

  given: "green"
